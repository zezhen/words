承接上一篇自我成长: 6年小总结, 本篇继续6年的总结, 主要侧重工作中的技巧.

和老板

和老板之间最本质的在于利益, 相互需要有可交换的资本, 谁都是有家有口的, 需要为自己谋福利. 而如何交换则是表象.
1) 有些人是老板的嫡系, 一起干过硬仗, 有深厚的’感情’, 这更多的是某种信任, 基于过去对对方未来的信心;
2) 有些人是老板的干将, 是老板依赖的能完成其重要任务的助手;
3) 有些人是老板的资源, 是老板协调项目的一个数字而已.

换位思考完, 想想作为员工, 希望从老板那得到什么? 事业指导, 资源整合, 升职加薪等等, 这些是目的. 如何取得是手段. 很多人会天真的以为老板都知道你做的事情, 等你够格了或者ready了, 他们自然会提供你需要的, 因为这很明显, 所有人都想升职加薪, 不言自明.

个人建议是做个”问题员工“, 有需求有问题要说出来, 说明白了! 记住会哭的孩子有奶吃! 如果你啥都不说, 或者不好意思说, 老板们就可以默认你一切都好, 无论真心或假意, 不提要求的员工最好’管’.

最良性最professional的关系是相互match expection! 达成某种利益交换的动态平衡.

some tips:

  1. 积极要求升职. 这是对自己负责, 最大化追求职业发展和经济收益. 不知是否达标? 可参考公司内部对级别的要求.
  2. 老板不同意升职. 那就问清原因gap等, 明确改进计划, 让老板提供你资源/项目以达成目的, 这是他的职责, 不要被轻松忽悠过去;
  3. Plan B. 万事不总顺利, 当第一个要求被拒绝, 可提出相对温和的请求, 比如加薪, 这样更容易被接受, 即所谓把门摔在脸上策略;
  4. 如果老板无法满足你的要求, 无论是因私人感情问题, 抑或是其能力不够, 对你来说都不够’好’, 可以考虑换一个了;
  5. 有些老板可能人很好, 且认为这是他的管理风格, 与人情无关, 人情太贵!
  6. 有人说我啥都不会, 没有谈的资本, 其实还是有的, 比如留你和招聘新人, 前者成本为X, 后者成本为Y, Y-X即为隐含的机会资本.

和同事

这里仅讨论工作关系, 且不谈衍生出的朋友关系. 工作中有合作也有竞争, 如果蛋糕足够大, 合作将多于竞争, 而蛋糕不够分的时候, 竞争则会凸显.

  1. 合作你能获得什么? 谨记切勿为他人做嫁衣.
  2. 专注自己核心竞争力. 锻炼自己技术[1], 巩固在公司的江湖地位[2], 其他次要的则拿多少credit做多少事.
  3. 若有利益冲突, 积极争取. 只有你自己对自己负责, 要强势! 大家都是成年人.
  4. 维持好关系. Be nice, build relationship! 少一个敌人/阻碍, 多一个朋友.

如何做项目

  1. WHY we need this project, what’s the Goal(s) or pain point(s)?
    • Finalize success criterion, system or metrics, prove it works
    • Need to understand the background/context.
  2. WHERE is the End State or ideal state?
    • The state that pain point(s) solved.
    • Use first principle, do the right things.
  3. WHAT is the Gap(s) from current state to end state?
    • Fully understand the current problem/position/tech-stack/etc
    • Data-driven, do analysis/investigation on the data
  4. WHICH we can solve based on current state? a.k.a. Scope
    • Need to consider tech-stack, priority, resource, etc
    • Set right expectation, make sure it matches with others
  5. HOW to reach the success? a.k.a. Execution
    • Breakdown projects into multiple phases and set timeline
    • Make it work, make it right, make it fast and make it repeatable
    • Update, update, update.

如何在公司成长

  1. Work more for larger scope and responsibility.
    • do not work for tasks only
    • always remember to ask WHY
    • show your vision and insight
    • work for your own territory
  2. Be confident and speak loudly.
    • do not work something silently
    • impress higher up with your achievement confidently
    • collect feedbacks and move forward
  3. Stop and see what others are doing.
    • look around to learn the environment around you, how important of your work
    • learn the ecosystem, enlarge your tech-width
    • prepare for next better projects or job
  4. Get some mentors/saviour[3], ask them to coach/help you
    • learn their experience, don’t repeat the wheel, find the shortcut
    • motivate saviour to help you on resource, promotion, information, etc

关于写Summary邮件

summary邮件to是一起合作的同事, 需要保证everyone’s on same page, cc才是最重要, 多数是大佬, 实际上邮件是给他们看的. 同时假设大佬们没有任何background. 所以邮件必须包含

  1. 简短的问题介绍, 明确说明要解决什么问题
  2. 现阶段进展如何, 我们采取了什么方案
  3. 下一步工作什么, 给大佬们以正确期望
  4. highlight blocker, call it out;
  5. 注意字体和排版.

关于Meeting

  1. 明白每一个meeting的目的, 自己的以及别人的, 勇敢说no;
  2. Meeting note很重要, 某种程度上是话语权, 结束前口头走一遍, 结束后邮件来一遍, 大家都很忙, 回顾只能看note;
  3. Meeting note中切记action items, 否则可能就白开了;
  4. 找老大review project, 要么是做了牛逼的东西, 需要被关注求credit, 要么有blocker, 需要support;
  5. 关注对方的肢体动作所表达的隐含意思, 因为多数人并不给明确回答 (还需要多练习!)

Best Practices

  1. 用好决策真值表. 当有两个或多个选择时, 列出所有决定因素, 横向比较;
  2. 准备文档. 用google doc记录工作内容, 比如各类设计, 执行流程, 总结和展望等等, 既能方便同事沟通, 老板review, 又能快速ramp up新人, 如果能为升职加码那就一举多得了;
  3. 记录Incidents. 记录下所有incidents, 发生原因, 解决方案等, 可能可以找到下一步优化方向, 撕逼时也可以用于自我保护或恶心他人.
  4. 文档短链接. 用短link链接各类文档或者项目目录, 比如Yahoo的yo link, 可自定义名字, 方便分享;
  5. 1分钟项目介绍. 快速向大佬们汇报工作, 比如在咖啡厅或电梯间, 你只有一分钟. 快速介绍背景, 可能的影响, high level解决方案, 项目重要成员或stackholder.
  6. 5分钟项目介绍. 面对manager或architect, 背景介绍, 解决方案(从大到小), 现阶段highlight进展, 遇到的问题, 下一步计划等.
  7. 学会delegate. 事情是做不完的, delegate给他人可能事半功倍, 人情就是这么攒起来的.
  8. 学会push back. 拒绝对你无益的要求, 比如各种杂活, 记得保持微笑; 或者以priority为理由, 放到backlog.
  9. …
